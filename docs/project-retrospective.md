# Project Retrospective

## Why this project exists

I kept this codebase as a small personal project because it covers a mix of practical frontend and product-thinking tasks that I enjoy:

- turning rough requirements into a usable UI,
- cleaning up inconsistent code,
- documenting tradeoffs,
- shaping simple business logic into something understandable.

## What I focused on

- clear table-driven data presentation,
- simple but typed full-stack flow with Nuxt,
- lightweight modal interactions,
- readable utility logic,
- incremental documentation.

## Refactor direction

To make the project feel less like a one-off exercise and more like an evolving product, I prioritized:

- removing challenge-specific language from the app and docs,
- tightening naming and behavior around food calculations,
- improving project documentation so it explains the app instead of the assignment.

## Future improvements

- Add filters for species, age range, and feeding needs.
- Persist animal data and feeding tasks with a real database.
- Add accessibility improvements for keyboard interaction in the table and modal.
- Add tests around the food-calculation rules and next-month aggregation.
